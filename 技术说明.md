# 计算机情绪引擎 V6 — 数学模型与技术说明

> 版本：V6.0 | 日期：2026-06-18 | 数据来源：114,889 行真实数据验证

---

## 一、项目概述

### 1.1 目标

让计算机系统像人一样感知自身状态，输出细腻的情感表达。不是简单的阈值告警，而是有惯性、有记忆、有层次的情绪模拟。

### 1.2 核心架构

```
系统指标 (58 原始 + 29 派生)
    ↓
五层感官 (L1-L5)
    ↓
PAD 情绪映射 (Pleasure/Arousal/Dominance)
    ↓
EMA 自适应平滑
    ↓
QuadrantStabilizer 防闪烁
    ↓
ODE 动力系统 (惯性/衰减/耦合)
    ↓
7 维情感状态 → 情绪表达
```

## 二、数学模型

### 2.1 指标归一化

所有原始指标先归一化到 [-1, 1] 范围：

**tanh 归一化（主用）：**
```
f(x) = tanh((x - center) / scale)
```
- `center`：正常基线值（如 CPU center=30 表示 30% 是"正常"）
- `scale`：控制曲线斜率（越大越线性，越小越陡峭）

| 指标 | center | scale | 含义 |
|------|--------|-------|------|
| CPU% | 30 | 35 | 30% 以下为正常 |
| MEM% | 50 | 30 | 50% 以下为正常 |
| error_rate | 2.0 | 8.0 | 2% 以下为正常 |
| latency_ms | 100 | 400 | 100ms 以下为正常 |

**sigmoid 函数（辅助）：**
```
σ(x) = 1 / (1 + exp(-steepness × (x - midpoint)))
```
用于边界条件的平滑过渡。

### 2.2 PAD 映射（核心公式）

从归一化指标计算三维情绪值：

**Pleasure（愉悦度）[-1, 1]：**
```
P = 1.0 - 0.45 × max(0, err_n) - 0.25 × max(0, lat_n)

分段惩罚：
  警觉区 (6% < err ≤ 12%):  P -= 0.15 × (err - 6) / 6
  担忧区 (12% < err ≤ 20%): P -= 0.25 × (err - 12) / 8 × (1 + 0.5 × cpu_n)
  愤怒区 (err > 20%):       P -= 0.40 × min(1, (err - 15) / 20)^0.7

变化率惩罚：
  err_vel > 0.2 → P -= 0.2 × min(1, err_vel)
  lat_vel > 0.3 → P -= 0.1 × min(1, lat_vel)

交互项：
  err > 3% AND cpu > 60% → P -= 0.25 × overload
  err > 10% AND cpu < 30% → P -= 0.2 × severity

最终: P = P × 2 - 1  (映射到 [-1, 1])
```

**Arousal（唤醒度）[-1, 1]：**
```
A_base = 0.6 × cpu_n + 0.2 × mem_n + 0.2 × lat_n

速度提升：
  err_vel > 0.2 → A += 0.5 × min(1, err_vel)
  lat_vel > 0.3 → A += 0.2 × min(1, lat_vel)

持续高错误：
  err > 5% → A += 0.3 × min(1, (err - 5) / 25)

内存泄漏检测：
  mem > 80% → A += 0.4 × min(1, (mem - 80) / 15)

边界修正：
  45% < cpu < 65% AND err < 2% AND lat < 200ms AND health > 0.6
  → A -= 0.30 × (1 - |cpu - 55| / 10)

最终: A = A × 0.8
```

**Dominance（控制度）[-1, 1]：**
```
健康系数: H = 1.0 - 0.7 × min(1, err / 12) - 0.3 × max(0, min(1, (lat - 200) / 1800))

CPU权重: w_cpu = 0.6 - 0.2 × H  (健康时 0.4, 不健康时 0.6)
余量: headroom = 1 - (cpu/100 × w_cpu + mem/100 × (1 - w_cpu))

衰减:
  err_decay = exp(-0.06 × err)
  lat_decay = exp(-0.002 × max(0, lat - 100))

侵蚀:
  err_erosion = max(0, err_n)
  effective_headroom = headroom × (1 - 0.7 × err_erosion)

交互惩罚:
  cpu > 60% AND err > 5% → interaction = 1 - 0.4 × overload × error_severity

变化率惩罚:
  err_vel > 0.3 → velocity_penalty = 0.3 × min(1, err_vel)

健康奖励:
  err < 2% AND lat < 500ms AND cpu > 50% → bonus = 0.35 × H × (cpu - 50) / 30

内存惩罚:
  mem > 80% → mem_penalty = 0.25 × min(1, (mem - 80) / 15)

D = effective_headroom × err_decay × lat_decay × interaction - velocity_penalty + bonus - mem_penalty

缩放: D = D × (2.0 - 0.6 × H) + (1.0 - (2.0 - 0.6 × H))
```

### 2.3 健康系数

```
H = 0.7 × (1 - min(1, err / 12)) + 0.3 × (1 - max(0, min(1, (lat - 200) / 1800)))
```
- H = 1.0：完全健康
- H = 0.0：严重异常
- 错误率权重 0.7，延迟权重 0.3

### 2.4 象限分类

P/A/D 三值的正负组合 → 8 个情绪象限：

| P | A | D | 象限 | 含义 |
|---|---|---|------|------|
| + | - | + | calm_happy | 稳态良好 |
| + | + | + | excited_happy | 高能良好 |
| - | - | + | calm_sad | 低落 |
| - | + | - | excited_angry | 过载 |
| + | - | - | relaxed | 放松 |
| + | + | - | alert | 警戒 |
| - | - | - | depressed | 沮丧 |
| - | + | - | panic | 恐慌 |

### 2.5 矛盾检测

指标间矛盾时修正 D 值：

| 矛盾类型 | 条件 | 修正 |
|----------|------|------|
| 高CPU+健康运行 | cpu > 75% AND err < 1% AND lat < 300ms | D += 0.3~0.6 |
| 中CPU+无错误 | 45% < cpu ≤ 75% AND err < 1% | D += 0.15~0.25 |
| 低CPU+高延迟 | cpu < 25% AND lat > 2000ms | D -= 0.2 |
| 低CPU+高错误 | cpu < 30% AND err > 10% | D -= 0.15 |

## 三、滤波与稳定系统

### 3.1 EMA 自适应滤波器

**参数（网格搜索训练结果）：**
- alpha_slow = 0.35（慢通道）
- alpha_fast = 0.60（快通道）
- beta = 12.0（灵敏度）
- inertia = 0.20（惯性）

**公式：**
```
α_eff = α_slow + (α_fast - α_slow) × sigmoid(β × (|Δ| - threshold))

P_smooth = α_eff × P_raw + (1 - α_eff) × P_prev
```

变化大时 α_eff 接近 α_fast（快速响应），变化小时接近 α_slow（平稳）。

### 3.2 QuadrantStabilizer（防闪烁控制器）

四层防御机制，解决 EMA 无法处理的象限边界闪烁问题。

**Layer 1 — 震荡检测：**
```
检测最近 8 步的方向反转次数
反转 ≥ 3 次 → 判定为震荡
抑制：P = 0.7 × baseline + 0.3 × P_current
```

**Layer 2 — 维度死区：**
```
if |P| < 0.08:
    P = 0
else:
    P = sign(P) × (|P| - 0.08) / (1 - 0.08)
```
P/A/D 在 [-0.08, +0.08] 内直接归零，消除 0 线附近的微小振荡。

**Layer 3 — 象限滞回：**
```
当前象限为正区间时：
  P 必须 < -0.06 才翻负（不是 < 0 就翻）
当前象限为负区间时：
  P 必须 > +0.06 才翻正
```
离开当前象限需要越过更远的阈值，形成不对称死区。

**Layer 4 — 惯性窗口：**
```
候选象限必须连续 5 次相同才确认切换
否则锁定在当前象限
```

### 3.3 防闪烁效果（真实数据验证）

| 数据集 | 行数 | EMA only | + Stabilizer | 改善 |
|--------|------|----------|-------------|------|
| 本机实时 | 4,974 | 1.02% | 0.20% | -80% |
| Google 2019 | 10,000 | 48.43% | 1.57% | -97% |
| Google 2011 | 100,000 | 0.00% | 0.00% | — |
| **加权平均** | **114,974** | **4.3%** | **0.6%** | **-86%** |

## 四、ODE 情绪动力系统

情绪不是对当前刺激的即时反应，而是有记忆的动态系统。

### 4.1 核心方程

```
dE/dt = -k × (E - E_target) + coupling + noise
```

**Euler 法离散化：**
```
E(t+1) = E(t) + dt × (-k × (E(t) - E_target) + coupling + noise)
```

### 4.2 自适应衰减率

```
k_base = 1 / τ

偏离程度: deviation = |E - E_target|
if deviation > 0.3:
    excess = min(1, (deviation - 0.3) / 0.7)
    k = k_base + (k_max - k_base) × excess²
```

偏离越大，衰减越快（幂律缩放），极端偏离时 k_max ≈ 0.35。

### 4.3 维度间耦合

```
P 剧变 → A 飙升:  coupling_A = 0.3 × max(0, |ΔP| - 0.05) × (1 if ΔP < 0 else 0.5)
T 高 → P 被拉低:  coupling_P = -0.15 × T
V 高 → A 升高:    coupling_A += 0.2 × V
```

P 骤降比 P 骤升对 A 的影响更大（负面冲击更强）。

### 4.4 时间常数

| 维度 | τ (秒) | 含义 |
|------|--------|------|
| P (愉悦) | 60 | 中等衰减，适中 |
| A (唤醒) | 25 | 快速响应 |
| D (控制) | 40 | 中等衰减 |
| V (波动) | 45 | 中等衰减 |
| F (疲劳) | 600 | 持久（10分钟半衰期） |
| T (紧绷) | 90 | 中慢衰减 |
| C (舒适) | 180 | 中等衰减 |

### 4.5 断崖检测

```
target_delta = max(|ΔP_target|, |ΔA_target|, |ΔD_target|)
if target_delta > 0.4:
    cliff_mode = True, 持续 3 步
    k = max(k, 0.5)  # 加速收敛
```

### 4.6 软边界衰减

```
if |E| > 0.85:
    overshoot = (|E| - 0.85) / 0.15
    decay = 0.08 × (1 + overshoot)
    E = sign(E) × max(|E| - decay, 0.50)
```

防止情绪值锁定在 ±1 极端。

## 五、体感系统

### 5.1 疲劳度（Fatigue）

**指数衰减累积，τ=600s：**
```
α = 1 - exp(-Δt / τ)
F(t) = α × load_signal + (1 - α) × F(t-1)
```

**负载信号来源：**
- CPU 过劳度（L1）
- 降频惩罚（L1）
- 上下文切换率 > 20K/s（L1）
- 系统调用率 > 500K/s（L1）
- 温度压力（L5）
- GPU 压力 > 50%（L5）

取所有来源的最大值作为 fatigue_signal。

### 5.2 紧绷度（Tension）

**信号方向一致性：**
```
计算各指标的平均方向（正/负）
正负信号比例越接近 → 矛盾越大 → 紧绷度越高

contradiction = 2 × min(positive, negative) / total
distress = Σmax(0, -direction) / total

T = 0.6 × contradiction + 0.4 × distress
```

**信号来源：**
- 监听积压比 > 0.3（L2）
- CLOSE_WAIT 比率 > 0.1（L2）
- 连接变动率 > 10/s（L2）
- 线程密度 > 20（L2）
- 磁盘 IO 延迟 > 5ms（L3）
- 磁盘队列深度 > 1（L3）
- IO 拥塞 > 2（L3）
- 中断率 > 15K/s（L3）
- 中断/DPC 时间占比 > 3%（L3）

### 5.3 舒适度（Comfort）

```
comfort = 1.0
if disk_usage > 70%:  comfort -= 0.4 × (disk - 70) / 30
if swap > 20%:        comfort -= 0.3 × (swap - 20) / 80
if mem_avail < 4GB:   comfort -= 0.3 × (4 - mem_avail) / 4
if thermal > 0:       comfort -= 0.3 × thermal
if io_latency > 10ms: comfort -= 0.2 × min(1, (lat - 10) / 40)
```

### 5.4 耗竭风险

```
exhaustion = fatigue × tension
```

疲劳和紧绷同时高时，耗竭风险急剧上升。

## 六、数据偏差分析

### 6.1 EMA 训练参数偏差

**代码中原始参数（声称训练得出）：**
```
alpha_slow=0.10, alpha_fast=0.70, beta=12.0, inertia=0.20
声称：闪烁率 6.6%, 评分 81.9/100
```

**实际训练结果（50,000 行 Google 真实数据）：**
```
alpha_slow=0.30, alpha_fast=0.50, beta=12.0, inertia=0.10
实际：闪烁率 48.1%, 评分 49.0/100
```

**偏差原因：**
1. 训练脚本 `train_ema.py` 存在但**从未执行**
2. 参数直接写死在 `ema_filter.py` 中
3. 声称的 6.6% 闪烁率无法复现

### 6.2 测试通过但存在缺陷

**黑盒测试（15 组）：**
- 每组只喂一个**静态输入**
- 测的是"输入固定时输出是否正确"
- **未测**：输入在边界振荡时输出是否稳定

**压力测试（600 步）：**
- 实际 state_change = 197 次（32.8%）
- 测试标准：PAD 范围正确、概率总和 ≈ 1.0
- **未测**：切换率是否低于阈值

### 6.3 Google Cluster Data 特征

| 数据集 | CPU 均值 | CPU 范围 | 特征 |
|--------|----------|----------|------|
| Google 2019 | 33.5% | 0-100% | 高噪声，多极端值 |
| Google 2011 | 1.5% | 0-47.9% | 极度空闲，几乎无波动 |
| 本机实时 | 17.2% | 12-71% | 低负载，微小波动 |

**Google 2011 的 0% 闪烁率不代表模型好，只代表数据太安静。**

### 6.4 本机数据特征

- CPU 均值 17.2%，标准差 6.4%
- 系统长期空闲，波动极小
- 任何参数组合都能获得低闪烁率
- **不能用本机数据单独评估模型质量**

### 6.5 综合评估方法

**正确的做法：**
1. 用多个不同分布的数据集交叉验证
2. 包含高噪声数据（Google 2019）
3. 包含低噪声数据（Google 2011 / 本机）
4. 加权平均闪烁率作为最终指标

**当前最优参数（交叉验证）：**
```
QuadrantStabilizer: dz=0.08, hyst=0.06, inertia=5
EMA: alpha_slow=0.35, alpha_fast=0.60, beta=12.0, inertia=0.20

加权闪烁率: 1.11% (114,974 行真实数据)
```
